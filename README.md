# Android技術課題

## 概要

株式会社ユニエイムのAndroid開発者向け技術課題です。
この技術課題は株式会社ゆめみがApache License 2.0で公開している[株式会社ゆめみ Android エンジニアコードチェック課題](https://github.com/yumemi-inc/android-engineer-codecheck/tree/06e32c7fe9879ad35d4b8e02688169fc805f30f0)をベースに作成しています。
アプリケーションのコードに関してはゆめみ社のものを踏襲していますが、具体的な取り組み方・観点・ストーリー等は独自のものに調整しています。

## 課題内容

### ストーリー

あなたはこのリポジトリのコードを今後3年間保守することを任されました。
3年間、毎日このコードを見るわけではありませんが、月に数回程度はこのコードを見て、数ヶ月に一回ぐらいは新しい機能をリリースするかもしれません。
そういった保守活動を最低3年間程度することを任されています。

### 設問

以下の課題すべてについて取り組んでください。取り組む順番は自由であり、 **実務を想像して適した順序で実施してください。**

- このコードを今後保守できることを示すために、何か簡単な追加機能を実装してください。
  - 〜数時間ぐらいで実装できるような機能で構いません
  - GitHubの機能を活用してください
- ストーリーに沿って、今後3年間、月に数回はコードを見る想定で、自分がメンテできるようにコードレビューをしてください。
  - 次のようなバグを含む箇所があります。
    - レイアウトエラー 
    - メモリリーク 
    - パースエラー 
    - 例外の処理漏れ 
    - クラッシュ
  - コードレビューの記述に関しては、開発者歴1〜2年ぐらいのやや経験の浅い開発者に説明することを想定してください。
  - レビューコメントはPRに記述してください。
- レビュー指摘に従ってコードを修正してください。
- テストを追加してください。

### 期限

1週間程度を目安として期日を連絡しますが、現在の就業状況等で取り組みやすい方・取り組みにくい方などいらっしゃると思いますので、過不足があれば遠慮なく申し出てください。
1日で終わらせる方もいれば、合間の時間をつかって2週間ほどかけて取り組むという方もいると思います。
この時間の使い方によって単純に優劣をつけることはありません。

### その他の補足

- このリポジトリはPrivateであり、試験を受ける方に対して1名に1つのリポジトリを作成提供しています。
- リポジトリをフォークする必要はありません。
- コードレビューや新機能の実装など、GitHubの機能を活用して、実際の開発を想定して取り組んでください。
  - GitHubの使用方法も評価対象となります。
- このリポジトリは課題終了後、1ヶ月を目安としてGitHub上で削除します。
  - 内容については自由に保存していただいて構いませんが、弊社からのフィードバックを直接的に公開することはお控えください。
- 作業結果を弊社でレビューするにあたって、Copilot等のAIサービスを利用する場合があります。
- AIサービスの利用は自由です。
  - 実際の業務においても、センシティブ・秘密性の高い情報を除いてAIサービスを利用することがあります。

## アプリ仕様

本アプリは GitHub のリポジトリを検索するアプリです。

<img src="docs/app.gif" width="320">

### 環境

- IDE：Android Studio Flamingo | 2022.2.1 Patch 2
- Kotlin：1.6.21
- Java：17
- Gradle：8.0
- minSdk：23
- targetSdk：31

※ ライブラリの利用はオープンソースのものに限ります。
※ 環境は適宜更新してください。

### 動作

1. 何かしらのキーワードを入力
2. GitHub API（`search/repositories`）でリポジトリを検索し、結果一覧を概要（リポジトリ名）で表示
3. 特定の結果を選択したら、該当リポジトリの詳細（リポジトリ名、オーナーアイコン、プロジェクト言語、Star 数、Watcher 数、Fork 数、Issue 数）を表示
